# Changelog

## v0.11.0 — Asymmetric bow support (yumi)

### Added

* Full support for asymmetric bow designs (e.g. Japanese yumi).
  The simulator now models the upper and lower limbs as fully independent
  members instead of mirroring half the geometry.
* New `nock_offset` field on `Draw` to place the nocking point off the
  geometric centre of the bow.
* Per-limb `Profile`, `Width`, `Layers`, and `Section` data on the input
  side, exposed in the GUI as separate top-level tree entries
  ("Profile (upper)", "Layers (lower)", etc.).
* Per-limb `Masses` (`limb_tip_upper`, `limb_tip_lower`, `string_nock`,
  `string_tip_upper`, `string_tip_lower`).
* New `RigidHandle` fields `length_upper` and `length_lower` (replacing the
  single `length`).
* New `BowInfo` output bundle exposing both `LimbInfo` halves
  (`upper`, `lower`) plus `pivot_point` and `nock_offset` at the bow level.
* `States` now carries lower-limb counterparts of every per-limb array
  (`lower_limb_pos`, `lower_layer_stress`, `string_tip_angle_lower`, …).
* New canonical `yumi.bow` example file shipped in
  `rust/virtualbow/data/examples/yumi.bow` and
  `docs/examples/bows/yumi.bow`, generated by
  `cargo run --release --example emit_yumi -p virtualbow`.

### Changed

* **File format break**: `.bow` files use a new schema (v5). Previous
  files are converted on load with a `.bak` backup written next to them.
* GUI: the 3D limb view now always renders the real asymmetric geometry;
  the legacy "view symmetric" toggle has been removed.
* Result-window plots and the model output grid expose `limb_mass_lower`
  and `nock_offset`, and source `pivot_point` from the bow-level common
  output rather than from the upper limb.

### Notes for asymmetric bows

The bracing search and several analytical consistency checks
(`draw_force = 2·string_force·cos(α)`, modal-damping target tolerances,
energy-balance derivative checks) still assume left/right-limb symmetry.
The example test suite skips these checks for `yumi*` files; users
modelling strongly asymmetric bows should expect a small residual force
in the braced state until the bracing solver is generalised.

## v0.9.1

### Fixed

* Wrong unit on string length result

## v0.9.0

### Added

* Configurable units for all inputs and outputs (including SI and US units)
* Easier way to model the profile using line, arc, spiral and spline segments
* Option to visualize the curvature of the profile
* Option to overlay images over plots
* Plot contents can be exported as CSV files
* Input fields for numbers accept arithmetic expressions
* Better error message on failed bracing

### Changed

* Different layout of the bow editor based on panels instead of dialogs
* Material properties are edited separately from the layer properties
* Improved monotonic cubic spline interpolation (width and heigh)
* Automatically convert the decimal comma to a dot for keyboard layouts that use a comma
* The number of layers is no longer limited (previously 15)
* The number of table rows is no longer limited (previously 100)

### Fixed

* Error when saving files to a path that contains special characters
* Missing error handling when loading and saving files
* Colors in the stress plot not always easily distinguishable

## v0.8.0

### Changed

* Main windows are now empty until a file is loaded or created

* Renamed 'storage ratio' to 'energy storage factor'
* Show minimum and maximum stresses instead of maximum absolute value
* The y-axis of the energy plot always starts at zero
* Increase resolution of images exported from plots
* New icons for the toolbar, menu actions and model tree
* Better random color generation for the layers
* Visual improvements to the limb view

### Fixed

* Action *New File* ignored modifications to already loaded files
* Remove files from *Open Recent* menu if they no longer exist
* Wrong default selection between static/dynamic results
